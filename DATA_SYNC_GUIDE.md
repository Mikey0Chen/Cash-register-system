# 财务统计数据同步说明

## 问题说明

当你在收银台完成结账后，切换到财务统计页面，**数据不会自动刷新**。

这是因为财务统计页面只在以下情况加载数据：
1. 页面首次打开时
2. 手动切换统计周期（今日/本周/本月）
3. 手动点击刷新按钮

## ✅ 解决方案

### 方法1：手动刷新（适用于生产环境 8081）

**步骤**：
1. 完成收银台结账
2. 进入财务统计页面
3. **点击"本周"或"本月"按钮**
4. **再点回"今日"按钮**
5. 数据就会更新了！

或者直接**刷新浏览器**（按 F5）

### 方法2：使用刷新按钮（开发环境 5173 已添加）⭐

**新功能**（仅开发环境）：
```
财务概览标题右侧 → 新增了一个 🔄 刷新按钮
点击即可刷新财务数据和订单明细
```

开发环境访问：http://localhost:5173/financial

## 🔧 技术原因

### 当前实现
```javascript
// 页面加载时获取一次数据
onMounted(() => {
  loadSummary()
  loadOrderList()
})

// 切换周期时重新加载
@change="loadSummary"
```

### 为什么不自动刷新？
1. **性能考虑**：频繁请求后端会增加服务器负载
2. **用户体验**：数据不断变化可能干扰用户查看
3. **网络消耗**：移动端流量消耗

### 未来改进方向
- 添加"刷新"按钮（开发环境已实现）
- 可选的自动刷新（每30秒/60秒）
- WebSocket实时推送（高级功能）

## ❓ 常见问题

### Q1: 为什么后端实时，前端不实时？
**A:** 前端是单页应用（SPA），数据存储在浏览器内存中。只有调用API时才会从后端获取最新数据。

### Q2: 能不能自动每10秒刷新一次？
**A:** 可以实现，但不推荐。频繁请求会：
- 增加服务器负载（2C2G服务器较弱）
- 消耗网络流量
- 影响页面性能

### Q3: 手动刷新麻烦吗？
**A:** 财务统计通常是阶段性查看（如下班前、每日/周/月结束），不需要实时监控，手动刷新完全够用。

### Q4: 其他页面会不会有同样问题？
**A:** 
- 收银台：无此问题（每次操作都是新请求）
- 配方管理：同样需要手动刷新
- 库存管理：同样需要手动刷新
